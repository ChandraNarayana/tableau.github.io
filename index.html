Contributing Code to Tableau Projects

Tableau projects accept pull requests (PR) for bug fixes and for new features. The process for each varies slightly:

Bug Fix

Before a PR will be accepted:

  • There must be an open, approved issue for tracking the bug.
  • The PR should contain a link to the bug's issue.

New Feature

To submit a new feature:

  • There must be an open, approved issue for tracking the new feature.
  • We will have an open discussion about the design.
  • New features will be accepted based on alignment with the project direction, impact (positive and negative), and serviceability.

Contributor License Agreement (CLA)

You will need to sign a CLA prior to any PRs being accepted. What is a CLA? It's a legal agreement that gives us permission to use the code you submit.

There are two CLAs: one for individuals and another for organizations.

To sign the agreement electronically, please email us and we'll send it to you (this is our preferred method ).
