Contributing Code to Tableau Projects

Many Tableau projects accept pull requests (check a given project's README) for bug fixes and for new features. The process for each varies slightly.

Bug Fix

Before a pull request (PR) will be accepted:

  • There must be an open, approved issue for tracking the bug.
  • The PR must contain a link to the bug's issue.
  • You must have signed a Contributor License Agreement (discussed below).

New Feature

Before you submit a new feature, make sure there's an open, approved issue for tracking the new feature.

We will have an open discussion about the design. New features will be accepted based on alignment with the project direction, impact (positive and negative), and serviceability.

Contributor License Agreement (CLA)

Before we accept a PR, you must have signed a CLA. What is a CLA? It's a legal agreement that gives us permission to use the code you submit. You only need to sign the CLA one time.

There are two flavors of CLA document: one for individuals and another for organizations.

To sign the agreement electronically, please email us and we'll send it to you (this is our preferred method ).
